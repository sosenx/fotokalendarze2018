File structure

We have used the BEM methodology for VSApp so in result you get a very intuitive file structure. Please click on the folders to discover what's inside.

VSApp file hierarchy
  • Expand all
  • Collapse all
  • documentation - template documentation
  • template - template files
    • assets - all common and vendor files used
      • fonts - font icons are stored here
      • img - used images
        • common - common images
        • vendor - images used by plugins
      • js - JavaScript files
        • common - common JavaScript functions
        • vendor - JavaScript libraries and plugins
      • less - LESS files
        • common - common less files like mixins, variable declaration, normalize etc.
        • vendor - LESS files required by plugins
    • blocks - all the blocks and their LESS, JS and PHP associated files
      • accordion - accordion block
      • box - box block
      • button - button block
      • carousel - carousel block
      • ... - etc.
    • dist - distributed CSS and JavaScript files
      • dev - development ready scripts used in the dev.html boilerplate
        • blocks.js - general function, bindings and plug-ins initializations.
        • layout.js - JavaScript layout used to wrap the blocks' scripts.
        • vendor.js - JavaScript libraries and plugins concatenated.
      • script.min.js - all the JavaScript files concatenated and minified
      • script.js - all the JavaScript files concatenated
      • style.min.css CSS code concatenated and minified
      • style.css CSS code concatenated
    • dev.html - development boilerplate that include unminified CSS and JavaScript files.
    • index.html - home page
    • index-2.html - secondary home page
    • index-3.html - tertiary home page
    • ... - etc
  • gulpfile.js - gulp tasks
  • package.json - Node.js dependencies

Getting Started

Using Gulp.js - VSApp comes with some basic Gulp configuration that should optimize and simplify your workflow. Please follow these steps in order to take full advantage of Gulp's build system.

  • 1

    Install Node.js.

    Go to nodejs.org and download the pre-built installer for your platform. Once it's downloaded, install it.
  • 2

    Open the Node.js command prompt (terminal) and install Gulp globally.

    Within the terminal please run npm install -g gulp. If you are on a Mac, you may need to run it with administrator permissions, so you'll have to run sudo npm install -g gulp instead.
  • 3

    Within the terminal, navigate to your project directory.

    Run cd path/to/your/project/, or you can open the terminal directly in the project's folder. Please note that you should navigate to the folder where the 'gulpfile.js' and the 'package.json' files are located, not to the 'template' folder.
  • 4

    Install the dependencies.

    Once you navigated to the project directory, run npm install and wait until all the required modules will be installed. Node.js is going to create a folder called 'node_modules' and inside that folder all the modules will be installed. Please check that folder for the following modules: browser-sync, gulp, gulp-clean-css, gulp-concat, gulp-less, gulp-rename, gulp-uglify, gulp-watch, gulp-wrap. If any of these are missing, then you'll have to install the missing ones manually. To do so run npm install [missing module], for example if 'browser-sync' is missing, then you'll have to run npm install browser-sync.
  • 5

    Run Gulp

    In the 'gulpfile.js' there are two Gulp tasks predefined - 'dev' task and 'prod' task. Please run gulp dev if you are in development and gulp prod if you finished the development process and want to move the project on the server. The difference between these two is that when you are in development mode gulp is going to watch files for changes and refresh the browser when needed, but in production mode it will concatenate and minify the JavaScript files. If you ran the gulp dev task, please don't close the terminal once all the sub tasks are finished because gulp is still watching for changes and this keeps the local server up and running. To stop the process in the terminal please press CTRL + C.
  • 6

    Start developing

    Since Gulp is watching for file changes, the browser will refresh each time you update any LESS, JavaScript or HTML file.
  • 7

    Edit blocks

    All the block's associated files are in the template/blocks/[block name]/ folder. Gulp watches for changes in those files and will rebuild when any changes are detected.

Without Gulp - VSApp has two predefined states: development and pre-production

By default all the files are in pre-production mode, that means HTML files include the concatenated versions of the JavaScript and CSS files. If you want to switch to development mode you can use the dev.html boilerplate. The dev.html file includes unminifed and separated JavaScript and CSS files. The minified versions of the CSS and JavaScript files are also available, so you may want to include them into your HTML file if you want to switch to the production state.

Block Structure

BEM stands for Block, Element and Modifier. According to the BEM methodology, a block is a logical and functionally independent page component. In VSApp every block has its folder inside the "blocks" folder, see below block's file structure.

  • blocks/
    • blockName/
      • blockName.less
      • blockName.js
      • blockName.php

Less Structure

See below the LESS pattern used to style VSApp.

copy 

.block{
	// Block styles
	text-align: left;
	
	&__element{
		// Element styles
		text-align: center;
	}

	&--modifier{
		// Modifier styles
		text-align: right;
	}
}

@color__secondary #1eb858 Main theme color.

@color__text--light #abafb8 Light text color.
@color__grey--light #f7f7f7 Light Grey used mostly for backgrounds.

@img__common--background "@{img__common}backgrounds/" Path to the background images.
@img__vendor "../assees/img/vendor/" Path to vendor images.

Logo

VSApp supports both logo types, image and text. In order to switch to the text logo, just replace the image with a text. By default the logo will switch to its dark version when the header becomes fixed, but if you want to prevent that, please change the 'data-switch' attribute to 'false', please note: this works only for images because texts can be restyled using pure CSS.

copy 
<!-- Logo Image -->
<div class="logo ">
	<a href="index.html" class="logo__link">
		<img src="path/to/logo" alt="Logo" class="logo__image js-logo__image" data-switch="true">
	</a>
</div>
<!-- End of Logo Image -->

<!-- Logo Text -->
<div class="logo">
	<a href="index.html" class="logo__link">
		Your Logo
	</a>
</div>
<!-- End of Logo Text -->

Icons

VSApp uses Fontello icon manager. For the full list of available icons, please visit Fontello's official page. Fontello allows you to create a custom collection of only needed icons without overloading the page with tones of other useless icons. In order to add more icons to the initial collection, please follow these steps.

  • 1

    Navigate to Fontello.com

  • 2

    Drag and Drop the 'assets/fonts/fontello/config.json' file on the Fontello.com page.

  • 3

    Fontello will automatically select the icons which were already in the collection so now all you have to do is to select the new icons you need.

  • 4

    Once you've added all the icons, press the "Download webfont" button.

  • 5

    From the downloaded archive copy and replace all the fonts in the "assets/fonts/fontello" folder also copy and replace the old 'config.json' with the new one which comes in the archive.

  • 6

    Copy all the content from the 'fontello-codes.css' file from the downloaded archive and paste (and replace) the old one from the 'assets/less/vendor/fontello-codes.less'.

  • 7

    Refresh your page and the new icons should be available. If you want to add some more icons later, just repeat these steps again.

copy 

<!-- Icon -->

<i class="fontello-facebook"></i>

<!-- End of Icon -->

Video

VSApp comes with a massive video block which consists of one big preview image and two secondary images behind, but since video itself is shown in a modal, you can simply add the 'js-video' class on any 'link' on the page and set its 'href' attribute to video's url and it will open that video in a responsive modal.

copy 

<!-- Video -->

<div class="video">
	<a href="[ Link to your video ]" class="video__holder js-video">
		<img src="[ Path to your Primary Image]" class="video__primary" alt="Primary Video Thumbnail">
		<img src="[ Path to your Secondary Image ]" class="video__secondary video__secondary--left" alt="Secondary Video Thumbnail">
		<img src="[ Path to your Secondary Image ]" class="video__secondary video__secondary--right" alt="Secondary Video Thumbnail">

Contact form

VSApp comes with a working contact form with smart data validation. You should configure it first, so please open the 'php/contact.php' file and set the '$myemail' variable to your specific email address where all the messages should be sent. The data will be wrapped by the included email template. Bellow you'll find the form's markup with its default validation attributes. For more information on data validation please visit the plugin's official documentation. The contact form will probably not work on your local server, so you may consider moving your project on your online server when testing it. Also, please make sure your 'php' folder has the 755 permissions set recursively.

copy 

<!-- Contact Form -->

<div class="contact-form  ">
	<form action="blocks/contact-form/contact-form.php" class="contact-form__form js-contact-form">
		<div class="contact-form__modal js-contact-form__modal">
			<h3 class="contact-form__message"> Your message was sent

			</h3>
		</div>

			<div class="col-md-12">
				<label class="contact-form__label" for="name-input">Your name</label>
				<div class="input input--contact  ">
					<input type="text" name="name-input" class="input__field" placeholder="" id="name-input" data-validation="required"> 
				</div>
				<label class="contact-form__label" for="email-input">Your e-mail</label>
				<div class="input input--contact  ">
					<input type="text" name="email-input" class="input__field" placeholder="" id="email-input" data-validation="required email">
